Finding Usability problems:


What is usability testing: It is the process of checking the people how they are using our design/creation/building with the main intention of.

 1)Make it easy.
  2)Proving that it is easy to use.

Do-it-yourself usability testing: Usability test are qualitative which are different from interviews and surveys which are quantitative. Its purpose is simple to enables us to know what we are building.

How it’s done: Basically, a facilitator and a participant are allowed to sit in a room. The participant was given with some task and the facilitator asked the him to think out loud when the user is doing that. Other set of people are watching the whole session though screen sharing software. At the end of the test observers had a debriefing session where they compared notes and discussed what are the problems faced and how they are going to fix that.

According to the author’s experience usability tests works even if it is a simple process, it reveals some critical issues.

• All most all sites have problems.
• All most all serious problems can be easily identified.
• Observing a user makes us a best designer.

Only few organizations use usability testing even though it is simple and valuable because.
• Most people don’t have any experience on usability testing.
• Lack of time for testing.
• Not willing to show the work until and unless it’s finished.

There is demo which demonstrate how to do usability testing to our own web site which will solve many of the critical problems. Link on the link below to watch 24 minutes of demo.
Demo: Go to www.rocketsurgerymadeeasy.com watch the “Demo Test” file.

Why most people don’t do usability testing: One of the main reason why most of the people don’t prefer doing usability testing because they think that it should be only done for big production- which is referred to as the Big Honkin’ test.

As per author suggestion a round of testing should be performed once in a month with three different users.

On the day of testing three different tests should be done by three different people in the morning and debrief over lunch. By this approach usability testing is done for that month and they can work on fixing the issues before next round of testing.

Do-it-yourself testing in agile: In agile methodology it is very touch to conduct usability test once in month because the life time of a cycle is too short. So, author suggest a new approach called “A morning a sprint”. In this process we will be testing the product more than once in a month with less number of participants and doing some amount of testing remotely which saves lot of time.

Average budget for a year of do-it-yourself testing: Approx. $4000-$6000
How to test other people sites: Give people some key inputs which are used to test our site. It’s better if each user works on 3 or more competitor sites. But at the time of debriefing instead of determining the worst things on site discuss the things which are good and which part of the website didn’t go well and from this what lessons can be applied to our web site.

Testing the sketch on the napkin: During early stage of planning of any project we would like to have a rough sketch of the project which will explain the project that the author refers as “sketch on the napkin”.

By showing this sketch on the napkin almost ask simple questions to everyone in the team by this we can understand whether our concept is easy to understand or not.

Testing wireframes: creating wireframes is followed by sketches in web design. Testing wireframes will not be time consuming because there are few people who can do it. Usually it is done in session which includes testing other sites and existing sites as well.

Testing page designs: Next step in wireframes is to create visual treatments of pages. Here wireframes focus on user interaction and visual design.

Who do you test with: Generally, in every team has tester who is familiar with the product. So, it’s better to hire people who is not familiar with the product so that it will be easy to check whether the end user is able to use you the application.

Why three users: Generally, first three users are mostly useful to find out defects and it is also easy when limited people are working on testing to find out the defects.

The easy way out: If we lack time and we can afford money it’s better to hire a recruiter. We can find lot of options online.

Heir and Spare: Having a substitute available all the time is very important. We have two options while recruiting people.

Just about anybody: It might me some other who is working in other department or the person who working for other company in same building.

• An “actual user” who we can test with remotely: We can hire a person who will be available on call according to requirement who can handle the situation remotely and able to join the at least for an hour.

Picking tasks to test and writing scenarios for them:

• First, we must choose the task which we are going to test and the things which the user is going to try.

• Then we should split the tasks into little scripts, scenarios and add details to the script which we want’s our user to know.

We should always have some extra task ready for the people who completes the work early. According to the author “filler” task is that the user must perform some actions on one of our competitor’s site.

Making the tasks into scenarios: Scenarios are the scripts which contains details of the task that a user can read and understand. We must phrase the sentences in such a way that it will be clear, easy to read and understand it without any confusion. We should try to keep the text in a simple way by not using uncommon words or unique words which appears on the screen.

Pilot test the scenarios: Once we are done with the scenarios we must do a pre-test which we call it as pilot testing which would take just 15 minutes.

Print them: Once we are done with all the final changes to the script we should print the whole script in two formats.

• One per sheet for the participants.
• All on one page for us and for observers.

Roles of a facilitator: As a guide we are going to tell the participants what to do and as a therapist we are going to encourage the participants to think this process is called as think loud protocol.
Things needed in the test room: We need to have a system with internet access, screen sharing and screen recording software.

Starting the session: Start the session by reading the script and which explains how our module is going to work. In every session we have pre-question at the start of the session and post -questions at the end of the session which helps every to participate in the session and we can understand whether everyone is able to follow the things which we are doing and by this we can grade the curve. We are going to ask the user to look at the home page and tell us what did they feel about the page.

The Tasks: At the start of every session we must handle a copy of the scenario to our participants and read it aloud. Repeat this for every scenario and we can clarify the participants if they have any. IF participants have more doubts we can ask them at the end. Once everything is done we can pay them and show them the door.

Prepare for the next test: For the next test we must complete clear our browse, delete the history and start a new screen sharing session. We must open a neutral screen for next test.
A few things to make everyone attend the session:

Make it easy for the people who are attending the session, advertise and make it clear that they are looking at it’s good if we provide the snacks to the participants.

Strategies for Fixing Usability problems:

The purpose of debriefing session has been clearly defined. By the time we come out of the door we need to have two things.
• A list of most major problems which occurred during usability testing .
• A set of problems which we are going to fix in next round of testing. We must fix most serious problems first.

How to run the debriefing meeting: The person who is going to run the test should run the debriefing session. Start the meeting with a glance of what we are going to do. We should ask everyone to go through the list of problem they have noted down and ask them to list any three problems which they feel as serious problem. Once they figure out the problems move around the room and ask the participates to read aloud. Finally, select ten problems out of these and based on the severity and discuss with the team how we are going to fix them. After the session it’s good to summarize everything which went in the session and in a form of short email.

The author suggests the below two principles for doing as little as possible:
• Tweak, don’t redesign.
• Take something away.

The Road Ahead:

Remote testing: This is the process where we reach our users remotely by screen sharing session and over phone.
By remote testing we have following advantages:
• Easier recruiting.
• No travel required.
• Easier scheduling.

Unmoderated remote testing: A service like “usertesting.com” can be used for unmoderated testing. One just must give them URL for what the test needs to be done and tasks. Provide them information about the kind of participants one is looking for and preferences like age group, computer experience, gender. Usertesting.com will put the request online to their pool of testers. Each user browses to the given URL and gives the test. Usertesting.com returns the screen recording of the participants.

 Chapter 1:

Don’t make me think:

Every web site which we are going designing should be self-explanatory. It should be very easy to get it without spending much time on it to understand. For instance, a person who is new to this web site should be able to easily figure out which button has which functionality he/she should be able to navigate easily to all the pages. Generally, representing the content with images is much preferred that writing the content inform of text which avoid lot of questions for the user.

Things that make us think:

All stuff in the website makes us to think unnecessarily. For example, unfamiliar technical terms. Every button should be designed in such a way that the user can easily understand what will happen if he clicks that button.

The website or the application which we are designing should be interactive and helpful for the user. For instance, when a user trying to search for a ting over the web site it should show up the things as suggestions related to that search which helps the user to search the tings quickly.

You can’t make everything self-evident:

It’s every tough to make everything self-evident on a web site but this can be achieved by using good colour combinations, text size and layouts. The most important note this that when we are not able to make that self -evident we should try to make the design as self-explanatory.

Why is all of this so important?

On the internet, the competition is too high when we frustrate our user then there are cent percent changes of losing our customers.

 How we really use the Web

SCANNING, SATISFICING, AND MUDDLING THROUGH

 Why are things always in the last place you look for them? Because you stop looking when you find them!

—CHILDREN’S RIDDLE

It is always different how people use the web site and how they actually use them.When we’re designing a web page we think that people will carefull read all the content witch is written on the web page .

What people does is they just give a glance at each page and scan the whole page where they can find some catchy key words or links.

FACT OF LIFE #1: We don’t read pages. We scan them.

Why do we scan?

• We’re usually on a mission: Most of the web use involves in getting something done so the users want to be done it quickly.

• We know we don’t need to read everything: Generally, users are not interested in everything which is written on the web page they look for the data which is useful for them so they scan the web page.

• We’re good at it:When we read a newes paper or a magazine we just skim the pages so most of people are good at doing that. They look for the data which is useful for them.

When we’re designing a web page, we assume that the content which we are updating in the site will be the best. But it is totally wrong we just update the stuff which come first to our mind and feel that it will be useful for the user.

Chapter 3:

DESIGNING FOR SCANNING, NOT READING:

We must keep following things in mind form user point of view how they will be scanning the page.

Conventions are your friends

Follow the web page conventions as user expects

Some of the basic things that a user expects is that the logo will be placed on the left hand top corner of the web page and if there is ant search bar it will be placed on the right hand top corner and also not only following web page conventions user should feel that it is wort using our webpage .

Create effective visual hierarchies:

One of the important thing is that the images which we are using in our web site should be eye catching and they should be large so that the user can easily observe the image.

Things that are connect with each other should be grouped under a same hood so that the user can easily identify the item for example in an ecommerce site all the workout product should be place under work out section in the left-hand menu list.

Cursor shape should be changed to hand from arrow once the cursor is moved on to the product and the color of the links should be differentiated from visited links to unvisited links.

One should take care while designing a home page if lot of stuff is kept on the home page user finds it to be too difficult to identify the things this is called clutter and every item which are placed on the webpage should be in good organized way.

Chapter 4:

Why users like mindless choices:

Generally, we find lot of choices over web. Making a choice on the webpage mindless feels good for the user. For instance, when I am trying to buy a product at my home office it shows an option where I can select the option whether I am buying the product for my home or for my office.

Chapter 5:

The art of not writing for the web:

Five or six things which author had learned in the college and he was stuck with one point which is E. B. White’s seventeenth rule in The Elements of Style

There are two specific kind of this which are useful while a website are:

• Happy talk must die.

• Instructions must die.

Happy talk must die: Happy talk is kind of self-congratulatory promotional which we generally find on bad written broachers. This writing says how good they are and what they did which is not considered as a useful information.

Instructions must die: Every site should be self-explanatory and instructions to the user must be eliminated completely. If in case we encountered with a situation where we have to put some instructions the they should be reduced to minimal.

Chapter 6:

Street signs and Breadcrumbs: Designing navigation

When we usually trying to find something, we have two options:

 1)To ask someone.

 2)To browse over internet.

If we choose second option we follow a hierarchy approach by following signs to guide us. Eventually, if are unable to find the thing which we are looking for we’ll leave.

When we decided to return to the website it is better to use conceptual hierarchy or retrace our steps instead of using physical sense. This is one of the main reason why bookmarks are used we can come back to the website which we had visited for our search.

Web navigation conventions: These are conventions which are generally used by all website. Generally, all users are used to this type of conventions. For instance, logo of a website is placed at left hand top corner and when we click on the log we are navigated to home page and search bar for a website is placed at right hand top corner. Web designers generally uses persistent navigation (or global navigation) to describe the set of navigations that appears on every page of website and one of the most important thing in global navigation is the button or link which navigates to thee home page.

Utilities are one of the most important things in a website such as contact us or help buttons which helps the user to find out what he/she is looking for.

There are four important things to be followed page naming.

• Every page should be named.

• The name should be placed at a right place.

• The name should be prominent.

• The name should be matched with what a user is clicking.

Enabling tab functionality to a web site is very good because it is self-evident, hard to miss, they’re slick and they suggest a physical space.

Chapter 7: The Big Bang Theory of Web Design

Main important thing for a website is home page. Every visitor gets an opinion on the website by looking at the home page. Home page should be designed in an orderly manner so that it will be easy for a visitor to get an idea about the website.

There are two important places on a page where we expect some statements which describes what is the site about.

• The tag line below the logo which is directly related to the logo. This is used as the description of the web site.

• The welcome blurb is the thing which is the description of the site, displayed in a prominent block on the Home page that’s visible without scrolling.

We should provide as much space as we can provide and we shouldn’t use mission statements as welcome blurbs.

Chapter 10: Mobile: It’s not just a city in Alabama anymore

Mobile phones being a handheld device have a small screen, one cannot show all the features that are available in the desktop version of the site. So a mobile-first approach must be followed. In mobile-first approach, only those features must be included which are most necessary for the user. One should also provide the desktop version must be provided for users who are familiar and comfortable using the desktop version. Nonmobile versions should allow zoom functionality, this makes user zoom and look for the information. 

If the mobile version has content in abundance it will take more time to load on the mobile device. This can frustrate a user. So mobile version should have content which is necessary. 

It can be a unique selling point of the application if it has some edge over the competition. A tutorial of how the application can be used when the application is launched the first time makes the application learnable. And its self-explanatory design makes it easily learnable if the user forgets how to use the app. 

To test a mobile application one can mount the camera on the phone. This will make it easy for a participant to use the phone naturally.
